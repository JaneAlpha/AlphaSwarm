## 核心 SQLite 数据库

路径：`D:\研一\ETF研究.tar\data\stock_data.db`

规模：约 73.9 MB

### 表：daily

用途：ETF 日行情主表。

字段：

```text
symbol, trade_date, open, high, low, close, volume
```

索引/约束：

```text
PRIMARY KEY (symbol, trade_date)
INDEX daily_symbol
INDEX daily_trade_date
UNIQUE INDEX daily_symbol_trade_date
```

当前观察：

- 行数：408,579
- ETF 数量：248 个 symbol
- 日期范围：2010-01-04 至 2026-05-29

### 表：fund_basic

用途：ETF 基础信息。

字段：

```text
id, symbol, name, index_symbol, index_name, list_date, fund_type
```

索引/约束：

```text
UNIQUE INDEX fundbasic_symbol
INDEX fundbasic_fund_type
```

当前观察：

- 行数：244

样例字段含义：

- `symbol`：ETF 代码，如 `510300.SH`
- `name`：ETF 名称
- `index_symbol`：跟踪指数代码
- `index_name`：跟踪指数名称
- `list_date`：上市日期
- `fund_type`：分类，如 `wide_base`、`industry`、`qdii`、`bond`

### 表：performance

字段：

```text
weekly_return, monthly_return, quarterly_return, yearly_return, sortino_ratio,
information_ratio, alpha, beta, win_rate, profit_loss_ratio, skewness,
kurtosis, total_days, positive_days, negative_days, created_at, updated_at
```

当前观察：

- 行数：244
- 覆盖 symbol：244 个
- 日期观察：`start_date` 最小为 2020-01-03，`end_date` 最大为 2025-12-19
- 注意：`end_date` 晚于当前行情面板探索中的部分日期结果，建议在新系统使用前重建或校验

## 新系统数据入口建议

优先级建议：

1. 首选读取 `data/processed/fund_selection_view.parquet`，用于 ETF 产品池、筛选、解释和推荐。
2. 首选读取 `data/processed/etf_daily_panel.parquet`，用于因子、择时、轮动、资金流和组合回测。
3. 使用 `data/processed/index_daily_panel.parquet` 和 `index_profiles.parquet` 做指数层分析。
4. 使用 `data/stock_data.db.daily` 做基础 OHLCV 回测或兼容旧 backtrader 数据入口。
5. raw 目录仅在需要重建 processed 数据或追溯单源数据时使用。
6. `analysis/` 和 `data/results/` 初期只做参考，不应成为新系统核心依赖。

## 新项目初始边界

新 ETF 智能策略系统建议先实现这些模块：

```text
src/
  data_client/       # 读取旧项目 SQLite/parquet 的只读数据接口
  universe/          # ETF 池筛选、分类、约束
  factors/           # 动量、资金流、估值、流动性、风险因子
  strategy/          # 策略信号与组合权重
  backtest/          # 回测与绩效
  agents/            # 智能策略解释、策略生成、研究助手
  app/               # 前端或命令行入口
```

当前新仓库仅保存本索引文档，后续代码开发再逐步加入。
